## Project Overview
`hara.lang` is a Lisp-like language and runtime built on the JVM. It provides an interactive development experience via a TCP server, allowing live code evaluation and inspection.

## Build & Test Setup
- **Build tool:** Maven (standard `pom.xml` configuration).
- **Java version:** 21 (as specified in the Maven compiler plugin).
- **Running tests:** `mvn clean test` executes the existing JUnit test suite located under `src/test/java`.
- **Coverage:** JaCoCo Maven plugin is already configured in `pom.xml`. After tests, run `mvn jacoco:report` to generate coverage reports under `target/site/jacoco`.

## Coverage Tooling
- **JaCoCo** provides line, branch, and method coverage metrics.
- The XML report (`target/site/jacoco/jacoco.xml`) can be parsed to identify uncovered classes/methods.
- The HTML report (`target/site/jacoco/index.html`) offers a visual overview.

## Lessons Learned
- **Maven Integration:** Using Maven simplifies dependency management and builds. The `maven-compiler-plugin` ensures consistent Java version across environments.
- **JaCoCo Usage:** Adding the JaCoCo plugin to the build lifecycle automatically instruments code during test execution, making coverage data readily available.
- **Testing Strategy:** Focus on unit tests for core language components (e.g., parser, evaluator, runtime). Mocking external resources (like the TCP server) keeps tests fast and deterministic.
- **Iterative Development:** The interactive TCP server enables rapid prototyping; however, comprehensive unit tests are essential to catch regressions before deployment.
- **Documentation Practices:** Maintaining a `GEMINI.md` file centralizes project knowledge, making onboarding easier for new contributors.
